Introduction To Nightingale

Getting Started

Welcome to the Nightingale Front-End Framework design documentation. Nightingale consists of a suite of design tools and helpers, all tailored to the specific brand image of the NHS. Use to information here to build consistent and brand-friendly pages.

Firstly, clone the framework repository from

https://github.com/NHSLeadership/nightingale

Before jumping into the Nightingale project, you must ensure that you compile the latest version of our CSS file, compiled straight from SASS. Compiling the main.scss file is easy!

  1. Open Terminal and cd to the Nightingale directory:

    $ cd /path/to/nightingale

  2. Run the SASS watch command

    $ npm run watch

    You might need to trick your machine into rebuilding the CSS for you. To do this, open main.scss and make any change at all, then save the file.

  3. That's all! You can now locate a compiled CSS file within the Nightingale directory.

Design Principals

Nightingale is based on a repsonsive grid-like layout system. It is built upon the CSS framework on 'tachyons', a simple technique in which any style can be built from a set of predefined classes containing predefined styling.

Using the 'tachyon technique' is simple. Just combine classes to give a desired final style.

Using this documentation, you must ensure that all brand guidelines are met. This ensures a unified image throughout the NHS, a vital aspect of improving the reliability and stability of the NHS brand.

Layout

The basis for any Nightingale page fundamentally begins with its layout.

The layout object (.o-layout) provides us with a column-style layout system. Items within the layout object are layout items (.o-layout__item). Introducing width utility classes changes the behavior of the items (e.g. u-1/2).

An exemplar layout scenario:

<div class = "o-layout"> <div class = "o-layout__item u-1/1"> </div> <div class = "o-layout__item u-1/1"> </div> </div>

Spacing

Spacing is used to nudge HTML around the page. It can be used to customise the positioning and spacing of any element. In its basic form, general spacing can be applied with two classes:

Margin:

.u-margin

Padding:

.u-padding

The utility class can then be appended to give closer control over where spacing is applied:

.u-[margin|padding]-[left|right|top|bottom]

Furthermore, the spacing can be changed in magnitude:

.u-[margin|padding]-[left|right|top|bottom]-[none|tiny|small|large]

For example, an element with class .u-margin-large would have large margin in every direction. Alternatively, an element with class .u-padding-top would have normal padding in the top direction.

Using breakpoints will introduce responsiveness to the grid system. Dependant on screensize, colums may collapse into a single column. Four breakpoints are avaliable:

Medium:@md

Large:@lg

Result:

A2

A3

Above, the system will occupy three columns whilst the screen-size is 'large' (@lg) or greater. Any size smaller than 'large', and the system collapes into one full width, and two half width columns.

If no fallback width class is explicitly stated, the default behavior will collapse the system to a single full width column. The following is an example where the default behavior is used:

Through careful testing, a responsive page can be built to suit any device.

Z-index

Use z-indexing to stack HTML elements on-top of eachother to build layered pages.

Access indexing using the .u-z-index-[0|10] class, and increase the value as approproate:

<div> <div style = "position:relative;" class="u-z-index-0 u-color-bg-nhs-blue">&nbsp</div> <div style = "position:relative;top:-15px;left:15px;" class="u-z-index-1 u-color-bg-nhs-orange">&nbsp</div> <div style = "position:relative;top:-30px;left:30px;" class="u-z-index-2 u-color-bg-nhs-purple">&nbsp</div> </div>

Result:

 
 
 

In order to further position the layered elements, items must have at least one position attribute, (relative, absolute ect.), and then use the [top|bottom|left|right] attribute to nude elements around.

Spinners

Spinners are used to signify a loading or waiting phase for a user. Use the .c-spinner class to style a Strong tag, ensuring that the phrase "Loading..." is written between the tags. If a browser does not support animation, the "Loading..." text will be displayed as a fallback.

Here is a basic spinner example:

<strong class = "c-spinner">Loading...</strong>

Ribbons

Ribbons are used to draw attention to key details at the very head of any document. Place a ribbon division before any other content within the main html body to ensure it appears at the head of the page. The structure of a ribbon consists of the following:

<div class = "c-ribbon"> <div class = "o-wrapper"> <div class = "c-ribbon__actions"> <button>Action</button> </div> <strong class = "c-ribbon__tag">[Tag Text Here]</strong> <strong class = "c-ribbon__body">[Body Text Here]</strong> </div> </div>

Result:

[Tag] [Body]

The ribbon consists of three key areas: The TAG, the BODY and the ACTION.

TAG:

.c-ribbon__tag (Optional) Provides detail to the overall message of the ribbon.

BODY:

.c-ribbon__body (Optional) Provides the key content of the ribbon.

ACTION:

.c-ribbon__actions (Optional) Provides an optional action, e.g. Close Ribbon. The button that lies within the actions division must be styled as a sprite, with an attached icon. Click to learn about sprites. It is possible to omit any of the optional areas to build a ribbon that suits your purpouse.

Ribbons can also be styled to give a visual cue as to their purpose. Append the following style varients to the outer ribbon division:

.c-ribbon--alpha:

Alpha This website is almost certainly in Alpha

.c-ribbon--beta:

Beta This website is almost certainly in Beta

.c-ribbon--live:

Live This website is almost certainly Live

Ribbons with expandable content can be included to include further details. To implement the expandability, use a details tag to hold the .c-ribbon__body class, with a nested summary inside. Click to learn more about details.

Here is an expandable ribbon example:

<div class = "c-ribbon c-ribbon--expandable"> <div class = "o-wrapper"> <details class = "c-ribbon__body"> <summary> <b>Expandable Content:</b> There is some really intersting expandable content below. Maybe you should expand the content and see what happens? How exciting! </summary> </details> </div> </div>

.c-ribbon--expandable:

Expandable Content: There is some really intersting expandable content below. Maybe you should expand the content and see what happens? How exciting!

Progress Bar

Progress bars are used to inform the user as to the progress of a process, or completion of an online form. The structure of the progress bar is:

<div class = "c-progress-bar"> <h3 class = "c-progress-bar__title">Progress Title</h3> <div class="c-progress-bar__bar"> <span class="c-progress-bar__fill" style="left: -50%;"></span> </div> <h3 class = "c-progress-bar__title-sub">Progress Subtitle</h3> </div>

Result:

Progress Title

Progress Subtitle

The progress bar consists of four areas: The TITLE, BAR, FILL & SUBTITLE.

TITLE:

.c-progress-bar__title Provides a title for the bar.

BAR:

.c-progress-bar__bar Provides a division container for the fill that spans the page.

FILL:

.c-progress-bar__fill Fills the division container dependant on the value of the style="left: [-100%|0%]".

SUBTITLE:

.c-progress-bar__title-sub Provides a smaller subtitle for the bar.

Progress Wheel

A variation of the progress bar, the progress wheel provides a lighter version of progress display:

<p class = "c-progress-wheel" data-percent = "33"> <b class = "c-progress-wheel__text">33%</b> </p>

Result:

33%

The progress wheel consists of two areas: The WHEEL & TEXT.

WHEEL:

.c-progress-wheel Provides a visiual progress wheel change the value through the data-percent = [0|100]" variable.

TEXT:

.c-progress-wheel__text Provides a text cue to the current progress value.

Pagination

Pagination is used to allow users to navigate around multiple pages within an enviroment. Pagination is styled from an 'ol' tag, and contains list items holding a link. Here is the basic structure of the pagination styling:

<ol class="c-pagination"> <li class="c-pagination__item c-pagination__prev"> <a href="#0" class="c-pagination__link"><span class="c-sprite c-sprite--prev-blue">Prev Page</span></a> </li> <li class="c-pagination__item"> <a href="#0" class="c-pagination__link"><span class="u-hide-visually">Page</span> 1</a> </li> <li class="c-pagination__item"> <a href="#0" class="c-pagination__link"><span class="u-hide-visually">Page</span> 2</a> </li> <li class="c-pagination__item"> <a href="#0" class="c-pagination__link"><span class="u-hide-visually">Page</span> 3</a> </li> <li class="c-pagination__item c-pagination__next"> <a href="#0" class="c-pagination__link"><span class="c-sprite c-sprite--next-blue">Next Page</span></a> </li> </ol>

Result:

  1. Prev Page
  2. Page 1
  3. Page 2
  4. Page 3
  5. Next Page

The first and last list items are arrow icons which are used to navigate directly to the previous and next pages within the page flow. Change the 'href' of the 'a' tag in order to change the targetted page. Similarly, change the number of .c-pagination__item styled list elements to reflect the structure of your site.

Note: The span tag with the hidden text is included as a fallback. Learn about display options here.
