Introduction To Nightingale

Getting Started

Welcome to the Nightingale Front-End Framework design documentation. Nightingale consists of a suite of design tools and helpers, all tailored to the specific brand image of the NHS. Use to information here to build consistent and brand-friendly pages.

Firstly, clone the framework repository from

https://github.com/NHSLeadership/nightingale

Before jumping into the Nightingale project, you must ensure that you compile the latest version of our CSS file, compiled straight from SASS. Compiling the main.scss file is easy!

  1. Open Terminal and cd to the Nightingale directory:

    $ cd /path/to/nightingale

  2. Run the SASS watch command

    $ npm run watch

    You might need to trick your machine into rebuilding the CSS for you. To do this, open main.scss and make any change at all, then save the file.

  3. That's all! You can now locate a compiled CSS file within the Nightingale directory.

Design Principals

Nightingale is based on a repsonsive grid-like layout system. It is built upon the CSS framework on 'tachyons', a simple technique in which any style can be built from a set of predefined classes containing predefined styling.

Using the 'tachyon technique' is simple. Just combine classes to give a desired final style.

Using this documentation, you must ensure that all brand guidelines are met. This ensures a unified image throughout the NHS, a vital aspect of improving the reliability and stability of the NHS brand.

Before beginning a layout, a wrapper division should be created to ensure proper edge margins and padding. Use the .o-wrapper class:

Layout

The basis for any Nightingale page fundamentally begins with its layout.

The layout object (.o-layout) provides us with a column-style layout system. Items within the layout object are layout items (.o-layout__item). Introducing width utility classes changes the behavior of the items (e.g. u-1/2).

An exemplar layout scenario:

<div class = "o-layout"> <div class = "o-layout__item u-1/1"> </div> <div class = "o-layout__item u-1/1"> </div> </div>

Spacing

Spacing is used to nudge HTML around the page. It can be used to customise the positioning and spacing of any element. In its basic form, general spacing can be applied with two classes:

Margin:

.u-margin

Padding:

.u-padding

The utility class can then be appended to give closer control over where spacing is applied:

.u-[margin|padding]-[left|right|top|bottom]

Furthermore, the spacing can be changed in magnitude:

.u-[margin|padding]-[left|right|top|bottom]-[none|tiny|small|large]

For example, an element with class .u-margin-large would have large margin in every direction. Alternatively, an element with class .u-padding-top would have normal padding in the top direction.

Using breakpoints will introduce responsiveness to the grid system. Dependant on screensize, colums may collapse into a single column. Four breakpoints are avaliable:

Small:@sm

Medium:@md

Large:@lg

X-Large:@xl

Append the breakpoint to the end of a width utility class. Here is an example situation. Resize you screen to see the effect:

<div class = "o-layout"> <div class = "o-layout__item u-1/1 u-4/12@lg"> <p>A1</p> </div> <div class = "o-layout__item u-6/12 u-4/12@lg"> <p>A2</p> </div> <div class = "o-layout__item u-6/12 u-4/12@lg"> <p>A3</p> </div> </div>

Result:

A1

A2

A3

Above, the system will occupy three columns whilst the screen-size is 'large' (@lg) or greater. Any size smaller than 'large', and the system collapes into one full width, and two half width columns.

If no fallback width class is explicitly stated, the default behavior will collapse the system to a single full width column. The following is an example where the default behavior is used:

<div class = "o-layout__item u-4/12@lg">

Through careful testing, a responsive page can be built to suit any device.
